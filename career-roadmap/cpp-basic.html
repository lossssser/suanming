C++ 基础语法

先把工程开发里真正会用到的 C++ 基础打牢，再往多线程、性能分析、平台开发和测试框架推进。

1. 语言基础 2. 对象与生命周期 3. 指针、引用、const 4. RAII 与资源管理 5. STL 常用容器 6. 现代 C++ 写法 7. 智驾工程练习

1. 语言基础

目标不是背语法，而是能读懂车端模块代码，知道变量、函数、类、命名空间、头文件和编译单元如何组织。

  • 掌握基本类型、作用域、函数声明与定义。
  • 理解 `.h` 和 `.cpp` 的分工，避免重复定义。
  • 熟悉命名空间、枚举、结构体、类的基本用法。
namespace ads {
enum class ModuleState { Init, Running, Error };

struct Header {
  uint64_t timestamp_ns{};
  uint32_t sequence{};
};
}

2. 对象与生命周期

C++ 工程里很多 bug 来自生命周期不清楚。要知道对象什么时候构造、什么时候析构、谁拥有它。

  • 构造函数、析构函数、拷贝构造、移动构造。
  • 栈对象、堆对象、静态对象的生命周期差异。
  • 所有权表达：值、引用、裸指针、智能指针。

3. 指针、引用、const

智能驾驶模块经常传递大对象、消息帧、感知结果和配置对象。引用和 const 能让接口更清楚。

  • `const T&`：只读传参，避免拷贝。
  • `T*`：可能为空，适合表达可选对象或输出参数。
  • `std::shared_ptr` / `std::unique_ptr`：表达共享或独占所有权。
void PublishFrame(const PerceptionFrame& frame);
bool LoadConfig(const std::string& path, Config* output);

4. RAII 与资源管理

RAII 是 C++ 工程的核心习惯：资源跟对象生命周期绑定，离开作用域自动释放。

  • 文件句柄、锁、socket、内存都适合 RAII。
  • 优先使用标准库对象管理资源。
  • 避免手写成对的 open/close、new/delete。
std::lock_guard<std::mutex> lock(mutex_);
UpdateSharedState();

5. STL 常用容器

先熟练掌握最常用的容器和算法，再去看复杂模板。

  • `std::vector`：连续内存，适合帧数据、点集、结果列表。
  • `std::unordered_map`：按 key 查询配置、状态、统计信息。
  • `std::deque`：适合滑动窗口、短历史缓存。
  • `std::algorithm`：排序、查找、过滤、转换。

6. 现代 C++ 写法

车端代码通常会使用 C++14/17。你需要熟悉工程里常见的现代写法。

  • `auto`、范围 for、结构化绑定。
  • `std::optional` 表达可能不存在的值。
  • `std::variant` 表达有限类型集合。
  • lambda 用于回调、排序、过滤。

7. 智驾工程练习

不要只刷语法题，要把语法放进智驾工程小任务里练。

  1. 写一个 `FrameBuffer`，保存最近 100 帧 HMI 状态。
  2. 写一个 `CoverageSummary`，读取覆盖率结果并生成统计。
  3. 写一个 `ModuleHealthMonitor`，跟踪模块状态和错误码。
  4. 给上述模块补 gtest 单元测试。
